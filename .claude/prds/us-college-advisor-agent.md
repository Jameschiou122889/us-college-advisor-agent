# PRD: US College Advisor Agent

## Executive Summary

打造一個 AI 美國升學選校顧問，幫助台灣 14-18 歲學生及家長快速獲得個人化的美國大學推薦清單。系統根據學生條件（GPA、標準化考試成績、興趣等）提供分層推薦（衝刺校/適中校/安全校），並在結果頁引導用戶預約真人顧問進行深度諮詢。

**價值主張**：
- 學生/家長：5 分鐘內獲得專業級選校建議，降低資訊不對稱
- 顧問團隊：AI 前端篩選提升效率，專注高價值諮詢服務

## Problem Statement

### 核心痛點
台灣學生申請美國大學面臨四大挑戰：
1. **選校困難**：3000+ 美國大學，不知道哪些適合自己
2. **資訊不透明**：錄取標準、學費、科系資訊分散難查
3. **Essay 撰寫**：不熟悉美式申請文書風格
4. **時間規劃**：複雜的申請時程容易錯過 deadline

### MVP 聚焦
根據顧問團隊經驗，**「我的條件能上哪些學校？」** 是學生和家長最高頻的問題。MVP 專注解決選校建議，後續再擴展其他功能。

### 市場機會
- 目標用戶：台灣每年約 5,000+ 學生申請美國大學
- 痛點驗證：現有顧問團隊已驗證需求存在
- 差異化：AI 提供即時、免費的初步建議，降低諮詢門檻

## User Stories

### Persona 1: 高中生小明 (17歲)
**背景**：高二學生，GPA 3.7，正在準備 SAT，對 CS 有興趣
**需求**：想知道自己的條件可以申請哪些學校
**使用場景**：
1. 進入網站，填寫基本資料（GPA、SAT 預估分數、想讀 CS）
2. 系統推薦 15 所學校（分衝刺/適中/安全三層）
3. 看到推薦結果後，點擊「預約顧問」了解更多

**Acceptance Criteria**：
- [ ] 表單填寫時間 < 3 分鐘
- [ ] 推薦結果在 10 秒內產出
- [ ] 每所學校顯示錄取機率和基本資訊

### Persona 2: 家長王媽媽 (45歲)
**背景**：女兒高一，開始規劃美國升學，想了解預算和選擇
**需求**：替女兒初步評估可行的學校範圍和費用
**使用場景**：
1. 輸入女兒目前成績和預算範圍
2. 查看推薦學校的學費和地區分布
3. 預約顧問討論完整規劃

**Acceptance Criteria**：
- [ ] 可篩選學費範圍
- [ ] 顯示每所學校年度費用估算
- [ ] 提供「家長專屬」諮詢選項

## Functional Requirements

### FR-1: 學生資料輸入表單

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| GPA | 數字 (0-4.0) | 是 | 支援 4.0 和百分制轉換 |
| 目標科系 | 下拉選單 | 是 | 提供 20+ 熱門科系選項 |
| SAT | 數字 (400-1600) | 否 | 可填「尚未考試」 |
| ACT | 數字 (1-36) | 否 | 可填「尚未考試」 |
| TOEFL | 數字 (0-120) | 否 | 可填「尚未考試」 |
| IELTS | 數字 (0-9.0) | 否 | 可填「尚未考試」 |
| 地區偏好 | 多選 | 否 | 東岸/西岸/中西部/南部/無偏好 |
| 學費預算 | 範圍選擇 | 否 | < $30k / $30-50k / $50-70k / > $70k / 無限制 |
| 課外活動 | 文字描述 | 否 | 自由填寫，AI 分析 |

### FR-2: 選校推薦引擎

**輸入處理**：
- 驗證必填欄位
- 標準化成績格式
- 處理「尚未考試」情況（使用預估值）

**推薦邏輯**：
```
錄取機率 = f(學生GPA, SAT, 學校錄取率, 學校SAT中位數, 科系競爭度)
```

**輸出分層**：
- **衝刺校 (Reach)**: 3-5 所，錄取機率 < 30%
- **適中校 (Match)**: 5-7 所，錄取機率 30-70%
- **安全校 (Safety)**: 3-5 所，錄取機率 > 70%

### FR-3: 結果展示頁

每所學校顯示：
- 學校名稱和 Logo
- US News 排名
- 錄取機率（高/中/低 標籤）
- 年度學費估算
- 熱門科系
- 地區位置
- 「了解更多」連結

### FR-4: 轉化機制

**CTA 設計**：
- 結果頁頂部：「想要更精準的分析？預約免費顧問諮詢」
- 每所學校卡片：「詢問顧問這所學校」
- 頁面底部：「下載完整報告 + 預約顧問」

**收集資訊**：
- 姓名
- 聯絡電話/LINE
- Email
- 期望諮詢時間

## Non-Functional Requirements

### 效能
- 推薦結果產生時間 < 10 秒
- 頁面載入時間 < 3 秒
- 支援 100 concurrent users

### 安全性
- 用戶資料加密儲存
- 不公開分享個人成績
- 符合個資法規範

### 可用性
- 支援繁體中文介面
- 響應式設計（手機/平板/桌面）
- 無需註冊即可使用基本功能

### 可擴展性
- 學校資料易於更新
- 推薦演算法可調整權重
- 可新增其他功能模組

## Success Criteria

### 關鍵指標 (KPIs)

| 指標 | 目標 | 衡量方式 |
|------|------|----------|
| 表單完成率 | > 60% | 開始填寫 → 提交 |
| 顧問預約率 | > 10% | 查看結果 → 預約 |
| 用戶滿意度 | > 4.0/5.0 | 結果頁問卷 |
| 推薦準確度 | > 80% | 顧問回饋驗證 |

### 成功定義
- MVP 上線後 30 天內獲得 100+ 顧問預約
- 用戶平均使用時間 > 5 分鐘
- 顧問轉化率較直接諮詢提升 20%

## Constraints & Assumptions

### 技術限制
- Phase 1 使用手動建立的 Top 100 學校資料庫
- 依賴 Claude API 穩定性
- n8n workflow 執行效能限制

### 資料限制
- 錄取率為公開統計數據，非即時
- 無法取得學校內部錄取標準
- 國際學生錄取率可能與整體不同

### 假設
- 用戶願意提供真實成績資料
- GPA 和標準化考試分數是選校主要依據
- 學生/家長傾向先免費試用再付費諮詢

## Out of Scope (Phase 1 不做)

- Essay 撰寫輔助
- 申請時間軸規劃
- 文件清單管理
- 獎學金搜尋
- 學校官網即時爬取
- 用戶帳號系統
- 付費訂閱功能
- 多語言支援（僅繁體中文）

## Development Tools

### n8n MCP Server
使用 n8n-mcp MCP Server 加速 workflow 開發：

| 工具 | 用途 |
|------|------|
| `search_nodes` | 搜尋可用節點（Form Trigger, HTTP Request 等）|
| `get_node` | 取得節點完整配置文檔 |
| `validate_node` | 驗證節點配置正確性 |
| `validate_workflow` | 驗證完整 workflow 結構 |
| `search_templates` | 搜尋相似 workflow 範本參考 |
| `n8n_create_workflow` | 直接部署 workflow 到 n8n instance |
| `n8n_test_workflow` | 測試 workflow 執行 |

**使用情境**：
- 開發新節點前，用 `get_node` 查詢正確參數格式
- 完成 workflow 後，用 `validate_workflow` 確認結構正確
- 部署前用 `n8n_test_workflow` 進行端對端測試

### Context7 MCP Server
使用 Context7 查詢最新 API 文檔：

| 工具 | 用途 |
|------|------|
| `resolve-library-id` | 解析 library 的 Context7 ID |
| `query-docs` | 查詢特定 library 的最新文檔 |

**使用情境**：
- 查詢 Anthropic Claude API 最新用法
- 查詢 n8n 節點最新配置方式
- 確保使用的 API 版本和參數是最新的
